Common language features

Like every programming language, SWRTJ implements some common features like the if-then-else statements, while statements etc.
Since SWRTJ is still a prototype, it doesn't implements all of them, for instance, arrays are no supported (but you can however use the CArrayList class).
The supported features are

  • Method and field expression chain: let 'x' be a variable or the this implicit parameter, you can type expression in the form x.f.m1(...).m2(...)... where 'f' is a field name (can be omitted) and 'm' a method name.
  • Default Types: SWRTJ includes a suite of java-like primitive types. Such types have the following subtyping relation
    byte subtype of short subtype of int subtype of long subtype of float subtype of double, char subtype of short and short subtype of char. The other primitive types are boolean and void.
    Moreover there is a suite of java classes such as String that is mapped to the IString interface and the CString class in SWRTJ.
  • Constants: the language supports numeric constants (0, 1, 2, ...) of type int, the null constant, the boolean constants true and false, and the string constants, in the form "t" where 't' is the text (a general character string).
  • Arithmetic expressions: where the operators +, -, *, /, % are supported with the same java semantics.
  • If-Else statement: the language supports this construct in the following form
    if(condition) {statement1} else {statement2} where the else branch can be omitted.
  • While: the language supports this construct in the following form
    while(condition) {statement1}
  • Conditions: conditions are boolean formulas allowed in if and while statements and in assignments.
    A condition is a list of atomic boolean formulas connected by the &, |, &&, ||, ^ operators (and, or, lazy and, lazy or, xor).
    An atomic formula is a boolean constant, boolean method and field expression chin or a comparison using the ==, != and numeric order comparation operators. In every condition C you can negate by typing !C.
